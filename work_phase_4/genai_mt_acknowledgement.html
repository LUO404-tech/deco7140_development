GenAI / MT Acknowledgement (WP4)

This page documents how Generative AI (ChatGPT) and MT-style prompts were used during work_phase_4. All outputs were reviewed, adapted to the WP3/WP4 style, and tested locally. The final website remains my own work.

1. Acknowledgement of GenAI / MT Use

I used ChatGPT as an implementation helper while building work_phase_4. It was mainly used to produce small, inspectable code and to clarify how to build specific WP4 interactions.

Course API helpers: generate JavaScript snippets to load community posts with GET and to submit new entries with POST.

Layout and image debugging: ask how to tune hero cards and form containers so they match the WP4 spacing system on desktop and mobile.

Feature-level guidance: ask how to add YouTube tutorial links on learning.html; how to build a “copy my contact email” button on support.html; how to set up a simple site search pattern on homepage.html; and how to implement the “Join the Community” inline-response behaviour on hubs.html (including which JavaScript structure to use).

I did not use AI to design whole pages, decide the site map, or pick the final visual style – those decisions came from my WP2 user stories, WP3 prototypes, and the A4 task sheet.

2. Completed GenAI / MT Usage Checklist

This section shows how my usage aligns with the UQ GenAI / MT framework (Week 10–11).

Clear, narrow prompts: I asked for small, auditable snippets (for example, “a minimal POST helper with headers and JSON handling”) instead of asking the AI to build the whole page.

Local testing: all generated code was reviewed, edited where needed, and tested in the actual WP4 pages before submission.

No personal data: prompts used placeholder emails / names and did not include student or private data.

Documented use: this page records where AI supported the build and where human decisions were kept.

3. Reflection on GenAI / MT Usage

Overall, GenAI helped me move faster on repetitive front-end tasks and gave me alternatives for DOM patterns I was not 100% confident about.

Creativity & problem-solving: it suggested different ways to attach a copy-to-clipboard action, to embed YouTube resources on the Learning page, and to show success / error messages after a POST.

Technical proficiency: I compared the AI suggestions with the course API docs and only kept the parts that matched the DECO7140 build.

Critical evaluation: some AI code overwrote or removed parts of my existing pages (for example, it restyled containers or replaced helper text). Because of that, I now always check the diff, keep my own version as the source of truth, and write more precise prompts so the AI does not touch layout or content that is already finished.

Future practice: I will continue to use AI for small, traceable blocks, but I will keep information architecture, content structure, and assessment text as my own work.

4. Integrity Statement

I am responsible for the final website. All AI-assisted code and copy were reviewed, localised to the project, and checked against the DECO7140 requirements. This page is provided to make my GenAI / MT usage transparent.

Authoring: I authored the final HTML, CSS, accessibility decisions, and page content.

Compliance: I followed the UQ GenAI / MT Usage Framework and the principles of academic integrity.
