GenAI Acknowledgement (WP4)

This document explains how Generative AI supported my WP4 work: integrating the course API, improving accessibility and CSS structure, and building a clean project portal. I remain fully responsible for design decisions, code quality, and compliance.

WP4 Focus

Where AI Helped

  • Shaping GET/POST modules and fetch patterns (getData.js, postFormData.js).
  • Designing an accessible filter chip pattern (ARIA and keyboard support).
  • Refactoring page-scoped CSS (remove overuse of !important, scope tokens).
  • Right-sizing hero/feature images for the News page.
  • Drafting the Project Portal layout and card microcopy.

AI generated drafts/snippets that I reviewed, edited, and tested locally.

Benefits

What It Accelerated

  • Faster iteration on API wiring and error handling.
  • Quicker convergence on accessible UI patterns (focus rings, touch targets, ARIA).
  • Cleaner CSS structure via page-scoped tokens and consistent naming.
Limitations

What It Didn’t Do

  • It did not decide site information architecture, sitemap, or visual style.
  • Generated code sometimes required fixes for edge cases and browser behavior.
  • Prompts needed specificity; I validated all outputs against assignment requirements and API docs.
Integrity

My Role & Accountability

I authored the final code and copy, verified accessibility (focus order, reduced motion, labels), and ensured the site meets WCAG 2.2 intent and project marking criteria. All API interactions follow the DECO7140 docs and use my own headers.

Concrete AI-Supported Contributions in WP4

  • Community Hubs — API Integration: guidance on structuring a small GET module and a POST helper using fetch, headers, and basic JSON handling; wiring success/error messages to the DOM.
  • Filter Chips: pattern for aria-pressed, keyboard activation (Space/Enter), and simple client-side filtering without heavy dependencies.
  • News Page CSS: compact hero icon sizing (object-fit: contain, max-height/width caps) and steady card image rhythm.
  • Introduction Page CSS: scoping accordion tokens to details.accordion, avoiding global :root leakage.
  • Project Portal: card grid semantics, concise microcopy, and navigation consistency with the rest of the site.

Decisions Made by Me

  • Information architecture and page content structure based on my Site Map and wireframes.
  • Final CSS tokens, spacing scale, and component polish to match the visual language across pages.
  • Accessibility checks (focus-visible, labels, reflow, touch target sizing, color contrast choices).
  • All copywriting, alt text, and selection of images/icons.

Prompting & Privacy

  • I asked for small, auditable snippets (e.g., “a minimal POST helper with form-data and headers”).
  • No personal or confidential data was sent to AI tools; example values were placeholders.
  • All outputs were manually reviewed, edited, and tested locally before inclusion.

Attribution

Generative AI assistance (ChatGPT) was used as a development aid for ideation, code scaffolding, and wording suggestions. I verified and adapted all outputs to meet the assignment requirements. The final site implementation is my own work.

Primary references during WP4: DECO7140 Community API docs (GET/POST), WCAG 2.2 principles, and course guidance.
